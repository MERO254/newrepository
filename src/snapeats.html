Key Insights

1. Firebase Backend Integration

Firestore Schema Design – Structuring collections for users, orders, and restaurants efficiently.

Real-time Updates – Using Firestore listeners for order status changes.

Security Rules – encrypting data

2. User Authentication & Security

Firebase Authentication – Email/Google login.

Data Security – Securing API keys, Firestore rules, and encrypted user data.

3. Payment Integration

M-Pesa API – Handling transactions securely.

Order Confirmation & Payment Status – Ensuring reliable transaction updates.

4. Efficient Data Loading & Performance Optimization

Pagination (Firestore Query) – Loading menus and orders efficiently.

Image Caching – Using Glide to optimize food image loading.

Offline Mode – Firestore’s offline persistence for seamless user experience.

5. Real-Time Order Tracking & Maps

Google Maps API – Implementing delivery tracking.

Live Order Updates – Using Firestore listeners for order status updates.

6. Notifications & User Engagement

Firebase Cloud Messaging (FCM) – Order updates, promotions, and reminders.

Push Notifications – Enhancing user retention and engagement.

7. Debugging & Testing

Crashlytics & Logcat – Monitoring crashes and errors.

Unit & UI Testing – Using JUnit and Espresso for reliability.
